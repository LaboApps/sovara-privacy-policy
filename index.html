Sovara works 100% offline. No internet connection is ever used. No account required. No server. No cloud. Your data never leaves your device — ever.

1. Overview

Sovara is a local Android application that monitors access to your device's microphone and camera by other applications. Unlike most security apps, Sovara requires no internet connection and sends absolutely nothing outside your device.

There is no backend server, no cloud storage, no analytics, no crash reporting, and no advertising network.

2. What data is stored

Sovara stores the following data exclusively on your device:

  • Timestamps of microphone and camera access events
  • Name and package identifier of the application that triggered the event (only if you grant the optional Usage Access permission)
  • Duration of each access event
  • Your in-app settings and preferences

This data is never transmitted anywhere. It exists only on your device and is fully under your control.

3. What data is NOT collected

  • No audio is recorded — ever
  • No photos or video are captured — ever
  • No location data
  • No device identifiers
  • No usage statistics sent to any server
  • No personal information of any kind

4. Permissions used

RECORD_AUDIO

Used exclusively to detect WHEN another application is accessing the microphone. Sovara never opens an audio recording stream, never captures audio data, and never processes audio content. We use this permission solely to query the system about microphone activity status.

You can verify this independently: Sovara's APK contains no calls to MediaRecorder.start(), AudioRecord.read(), or any audio capture method. The permission is required by Android because querying microphone state from a background service falls under audio permissions, even when no audio is captured.

PACKAGE_USAGE_STATS

Optional. Used to identify which application was in the foreground when a microphone or camera access occurred. Must be granted manually by the user in Android system settings. Sovara works without it — app names may simply appear as unknown.

FOREGROUND_SERVICE / FOREGROUND_SERVICE_MICROPHONE

Required to run the monitoring service continuously in the background. Android requires a visible notification when a foreground service is active.

Camera monitoring — no CAMERA permission required

Sovara monitors camera usage via the Android system API CameraManager.AvailabilityCallback. This API notifies when any camera becomes available or unavailable (i.e., in use). Sovara does NOT require or request the CAMERA permission — it never captures images or video.

POST_NOTIFICATIONS

Used to display real-time alerts when microphone or camera access is detected.

RECEIVE_BOOT_COMPLETED

Used to restart the monitoring service automatically after device reboot.

VIBRATE

Used to vibrate the device as part of alert notifications.

5. No third-party services

Sovara contains zero third-party SDKs that transmit data externally. No Google Analytics, no Firebase, no Crashlytics, no Facebook SDK, no advertising network of any kind.

6. Data deletion

  • Delete history: History screen → Delete button
  • Uninstall: Removes all data permanently

7. Children

Sovara does not collect any personal information from any user, including children.

8. Contact

For any questions, contact the developer via the Google Play Store listing page.
